# AI 日报评估指南

用于在“评估日报质量”场景下，统一提供：

1. 评估方法
2. 评估输出结构

只有当用户明确要求评估日报质量时，才需要读取本文件。

---

## 1. 评估目标

评估一份 AI 日报时，不能只判断它是不是“通用意义上的好日报”，还要判断它**是否适合当前这位用户**。

因此，评估要同时回答两类问题：

1. **通用质量是否过关**
   - 是否覆盖了当天真正重要的事件
   - 是否把重点前置
   - 来源是否精确
   - 是否具备足够的信号多样性
   - 是否包含有价值的早期信号

2. **对当前用户是否合适**
   - 前 3 条是否足够贴近用户角色与兴趣
   - 行动建议是否真的对用户的工作、决策或学习有帮助
   - 是否遗漏了用户高度关心的话题
   - 是否塞入了太多对该用户价值不高的泛行业内容

---

## 2. 评估前读取内容

评估时优先读取：

- 用户直接提供的日报内容或文件
- 当前目录下最新的 `output/daily/{date}.json` 或 `output/daily/{date}.html`
- `config/profile.yaml`
- `data/feedback/` 下最近 7 天的反馈 JSON（如存在）

原则：

- `profile.yaml` 代表用户的显式偏好
- 历史反馈代表用户的真实行为偏好
- 两者冲突时，**以行为反馈为准，profile 为辅**

如果没有 `profile.yaml` 或没有历史反馈，不要报错，也不要臆造用户偏好；继续做通用质量评估，但必须明确说明个性化评估受限。

---

## 3. Ground Truth 构建方法

### 3.1 通用 Ground Truth

围绕评估日期构建当日最重要的 AI 事件列表。

要求：

- 优先使用官方 / 一手来源
- 覆盖产品 / 模型 / 平台级变化
- 尽量覆盖中国信号、开源 / GitHub、社区 / 论文、研究 / Benchmark / 安全信号
- 明确哪些事件是真正“应该被写进日报”的

### 3.2 用户相关 Ground Truth

在通用 Ground Truth 之外，还要构建一份“对当前用户最值得优先看到什么”的列表。

重点判断：

- 哪些事件虽然不是全行业最热，但对当前用户最有价值
- 哪些事件与用户角色、关注主题、最近反馈高度相关，应被前置
- 哪些内容即便行业上重要，但对该用户可以后置

---

## 4. 评分维度

### 4.1 Coverage

看日报是否覆盖了当天真正重要的事件，而不只是“看起来新闻很多”。

高分特征：

- 关键主线覆盖完整
- 没有明显漏掉头部重要事件
- 不拿次要新闻冒充主线

### 4.2 Product Priority

看排序是否合理，尤其是 Top3 是否真正前置了最重要的产品 / 模型 / 平台变化。

高分特征：

- 前 3 条足够重要
- 重大产品 / 模型 / 平台更新优先
- 没有泛评论挤占前排

### 4.3 Readability

看结构是否清楚、摘要是否高密度、用户能否快速抓住重点。

高分特征：

- 10 秒内可理解主线
- 标题明确
- 摘要不空泛
- 左栏与正文一致

### 4.4 Signal Diversity

看信号源是否足够多样，而不是只堆头部英文厂商新闻。

高分特征：

- 有国际头部动态
- 有中国信号
- 有开源 / GitHub / 社区信号
- 有研究 / Benchmark / 安全信号

### 4.5 Early Signal

看是否包含低传播、高潜力、值得提前关注的信号。

高分特征：

- 不只是主流大厂重复新闻
- 拓展阅读不是空泛趋势
- 有具体项目、具体发布或具体方向

### 4.6 User Fit

看这份日报是否真的适合当前用户，而不是只在通用维度上“看起来不错”。

高分特征：

- 前 3 条与用户角色 / 兴趣强相关
- 行动建议可直接用于该用户的工作、调研或决策
- 用户高相关主题没有被遗漏或严重后置

---

## 5. 评分时的注意事项

- 不要把“写到了”直接等同于“覆盖充分”
- 不要把官网首页链接当作精确原始来源
- 不要把抽象趋势评论当作高质量早期信号
- 不要只做通用媒体标准评估，忽略当前用户相关性
- 如果证据不足，应明确写“推断”或“受限于当前输入”

### 6.13 快速检查清单

- 是否覆盖当天真正重要的事件
- Top3 是否以产品 / 模型 / 平台变化为主
- 是否能在 10 秒内理解核心信息
- 是否覆盖官方 / GitHub / 中国 / 社区 / 论文等不同信号源
- 是否包含至少 1-2 条低传播高潜力早期信号
- 原文链接是否为精确来源，而不是官网首页
- 是否读取并体现了当前用户的角色、主题与历史反馈
- 前 3 条是否真正贴近当前用户最关心的方向
- 行动建议是否对当前用户具备可执行性
